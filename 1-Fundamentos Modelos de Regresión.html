Aprendizaje Supervisado¶

Definici√≥n: Permite realizar predicciones basado en los datos hist√≥ricos de la organizaci√≥n. En este caso a diferencia del aprendizaje no supervisado se cuentan con ejemplos que permiten identificar el comportamiento de los datos. Se compone de dos variables:

  • Target Variable: Es la variable que desea ser predecida. Tambi√©n son conocidas como las variables dependientes

  • Predictor Variables: Corresponde al conjunto de variables que utilizar√° el modelo para predecir la variable target. Tambi√©n se les conoce como Features o variables independientes.

Casos de Usos

  • Predecir valores de conjuntos de datos desconocidos. Como los precios de viviendas, el costo de autom√≥viles, el precio los alquileresy otro tipo de variables continuos.

Cross Validation¶

Corresponde a una t√©cnica utilizada para la evaluaci√≥n de resultados de modelos de regresi√≥n que nos permite estandarizar el rendimiento, mediante la divisi√≥n de los datos en bloques de conjuntos de trainning y testing de una manera autom√°tica. Esta t√©cnica permite garantizar que los modelos son independientes de la partici√≥n entre datos de entrenamiento y prueba.

Consiste en repetir y calcular la media aritm√©tica obtenida de las medidas de evaluaci√≥n sobre diferentes particiones. Se utiliza en entornos donde el objetivo principal es la predicci√≥n y se quiere estimar la precisi√≥n de un modelo que se llevar√° a cabo a la pr√°ctica.A diferencia de la separaci√≥n tradicional de datos en Trainning y Testing, con Cross Validation se elimina la dependencia de los resultados basados en la selecci√≥n del conjunto de datos.

En esta t√©cnica se utiliza en dividir el conjunto de datos original en K particiones distintas(tambi√©n llamados Folds), donde una de ellas ser√° el conjunto de datos de Prueba y el resto de Entrenamiento, luego se realizan K iteraciones hasta haber utilizado cada uno de los bloques como conjuntos de Prueba. Una vez que finalizan las iteraciones se calcula una media aritm√©tica o promedio de los resultados obtenidos. Este proceso se puede observar en la imagen siguiente:

Regularización Ridge¶

La regularizaci√≥n Ridge penaliza la suma de los coeficientes elevados al cuadrado. A esta penalizaci√≥n se le conoce como l2 y tiene el efecto de reducir de forma proporcional el valor de todos los coeficientes del modelo pero sin que estos lleguen a cero.

Casos de Uso

  • Es recomendable usarlo cuando es necesario evitar el sobreajuste. Debido a que incluye todas las funciones o coeficientes, no es muy √∫til en el caso de que el conjunto de datos cuente con un n√∫mero de caracter√≠sticas exorbitantemente altas, ya que plantear√° desaf√≠os computacionales por la capacidad de procesamiento requerida.

Regularización Lasso¶

La regularizaci√≥n Lasso penaliza la suma del valor absolutos de los coeficientes de regresi√≥n. A esta penalizaci√≥n se le conoce como l1 y tiene el efecto de forzar a que los coeficientes de los predictores tiendan a cero. En este caso Lasso permite seleccionar los coeficientes menos relevantes y los hace cero.

Casos de Uso

  • Conviene utilizarlo cuando no todos los coeficientes son relvantes para el modelo y se requiere analizar una cantidad muy grande de informaci√≥n.
